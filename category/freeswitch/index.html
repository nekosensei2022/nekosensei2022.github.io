FreeSWITCH ESL 开发小结

最近用 Go 语言开发了一个基于 FreeSWITCH 的小模块。

功能很简单，利用 inbound 模式的 ESL 来实现呼叫接续。这里记录一下一些关键的点，以防忘记。

相关 FS 命令

本次开发，主要是用 ESL 来收取事件，获得呼叫的状态，然后通过命令，向 FS 下达指令，从而达到控制呼叫的目的。

用到的命令主要有：

originate

用于发起 B 侧呼叫。

用法类似这样：

originate {origination_caller_id_number=111}sofia/gateway/test/222 &park

FreeSWITCH 集成百度 MRCP 产品

部署百度 MRCP

参考此文档所述之步骤。

  1. 注册百度开发者账号，创建应用

    应用管理控制台的地址： https://console.bce.baidu.com/ai/#/ai/speech/app/list

  2. 下载 MRCP 服务端

    https://ai.baidu.com/download?sdkId=111

  3. 修改服务端配置文件

    1. /mrcp-server/conf/unimrcpserver.xml

      修改unimrcpserver->properties->ip为本机 IP 地址。

    2. mrcp-server/conf/mrcp-asr.conf

      修改AUTH_APPID和AUTH_APPKEY为开发者AppID和API Key。

    3. mrcp-server/conf/unimrcpserver_control.conf

      修改./bin/check 127.0.0.1 1544中的127.0.0.1为本机 IP 地址。

  4. 设置 gcc 环境

    sudo ./bootstrap.sh

  5. 测试运行

    cd mrcp-server
./bin/unimrcpserver -r . &

  6. 客户端验证

    1. 将必要库放入LD_LIBRARY_PATH环境变量

      export LD_LIBRARY_PATH=${SERVER_ROOT}/mrcp-server/lib:$LD_LIBRARY_PATH

      其中${SERVER_ROOT}替换为真实路径。

    2. 进入客户端交互环境

      cd bin
./asrclient

    3. 输入指令

      run grammar.xml xeq.pcm

      可以看到客户端与服务端的交互过程，如果成功，则会输出类似这样的识别结果：

      <speech-to-text confidence="100">[17daae42b8f34bac_2_1]我要去西二旗我要去西二旗</speech-to-text>

  7. 启动守护进程

    如果上述操作成功，则可以将 MRCP 启动为守护进程。

    cd mrcp-server
./bin/unimrcpserver_control start

FreeSWITCH 命令行客户端非本地访问

FreeSWITCH 安装后，默认配置下，命令行客户端（fs_cli）只能从本地访问。

如果从其他机器的 fs_cli，是连不上的：

[ERROR] fs_cli.c:1699 main() Error Connecting []
